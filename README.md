# Сервис бронирования переговорных комнат (Meeting Room Booking Service)

Асинхронный веб-сервис на базе **FastAPI**, разработанный для автоматизации бронирования переговорных комнат в коворкинге. 

## Технологический стек

* **Язык разработки:** Python 3.12.3
* **Фреймворк:** FastAPI (Async/Await архитектура)
* **СУБД:** PostgreSQL 15 (основная база), SQLite In-Memory (для изолированных тестов)
* **ORM:** SQLAlchemy 2.0 (асинхронный драйвер `asyncpg`)
* **Валидация данных:** Pydantic v2
* **Менеджер зависимостей:** Poetry
* **Тестирование:** Pytest + Pytest-Asyncio + HTTPX
* **Оркестрация:** Docker / Docker Compose (с поддержкой Healthcheck для СУБД)

## Как запустить проект

Для запуска проекта вам понадобятся только установленные **Docker** и **Docker Compose**. Всё окружение разворачивается автоматически.

1. **Клонируйте репозиторий и перейдите в папку проекта:**
   ```bash
   git clone <ССЫЛКА_НА_ВАШ_РЕПОЗИТОРИЙ>
   cd meeting-room-service
   ```

2. **Запустите сборку и поднимите контейнеры:**
   ```bash
   docker-compose up --build
   ```
   *Контейнер с FastAPI (`web`) автоматически дождется, пока PostgreSQL (`db`) пройдет проверку здоровья (`healthcheck`) и будет полностью готов принимать асинхронные соединения.*

3. **Интерактивная документация API:**
   После успешного запуска откройте браузер по адресу:
   👉 [http://localhost:8000/docs](http://localhost:8000/docs) (Swagger UI)

## Правила авторизации (Эмуляция заголовков)

В соответствии с техническим заданием, роли и идентификаторы пользователей передаются через HTTP-заголовки (Headers):
* `x-user-id` (string) — уникальный ID сотрудника (например, `emp_10`). Обязателен при бронировании и отмене.
* `x-role` (string) — роль пользователя (`employee` по умолчанию или `admin`).

## Запуск интеграционных тестов

Тесты полностью изолированы от основной базы данных. При запуске тестов создается сверхбыстрая временная база данных **SQLite прямо в оперативной памяти** (In-Memory), которая очищается после завершения сценария.

Для запуска тестов внутри активного Docker-контейнера выполните команду:
```bash
docker exec -it booking_fastapi_web poetry run pytest -p no:cacheprovider
```

## Реализованные эндпоинты

* `GET /rooms/availability?booking_date=YYYY-MM-DD` — Просмотр доступных (свободных) временных слотов для всех комнат коворкинга на выбранную дату.
* `POST /bookings` — Создание бронирования сотрудником на конкретную комнату, дату и фиксированный слот.
* `DELETE /bookings/{booking_id}` — Отмена бронирования. Обычный сотрудник может удалить **только свою бронь**. Пользователь с заголовком `x-role: admin` имеет право удалить **любое** бронирование.
